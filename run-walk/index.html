Run/Walk Data Pipeline

A small, production-minded batch pipeline that ingests CSV telemetry, converts to Parquet, and exposes fast queries via DuckDB. Designed and built for reproducible analytics and low-cost storage.

Project description

This project ingests raw CSV export files from devices, validates and normalizes fields, converts them to partitioned Parquet, and maintains materialized DuckDB tables for fast analytical queries and dashboards. The pipeline is orchestrated with Apache Airflow and the dashboard is a lightweight Streamlit app used by analysts.

Key features

  • Incremental batch ingestion with idempotent transforms
  • Row-level validation and schema checks with automated alerts
  • Parquet output for compact, columnar storage
  • DuckDB materialized tables for sub-second analytical queries
  • Lightweight dashboard and REST API for ad-hoc reporting

Impact

  • Enabled daily materialized reports that analysts can query in seconds instead of waiting for slow CSV loads.
  • Significantly reduced storage and I/O by converting raw CSV to compressed Parquet, lowering ongoing storage costs and speeding downstream processing.
  • Turned manual reporting workflows into automated APIs and dashboards, cutting analyst time to insight from hours or days down to minutes.
  • Improved observability with validation alerts and monitoring so data quality issues are caught earlier and incidents are resolved faster.

How to run locally

  1. Clone the repository: git clone https://github.com/mrprogrammingx/run-walk-pipeline.git
  2. Install dependencies (Python) and start Airflow in LocalExecutor or use Docker Compose.
  3. Set storage path for Parquet outputs (local or cloud) and run the DAG.

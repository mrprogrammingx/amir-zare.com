Real-Time Chat Application

A classic, production-minded web chat application built with FastAPI and SQLite (WAL) supporting real-time messaging, presence, and multi-device sessions. Designed for privacy, reliability, and simple deployments.

Project description

This project implements a feature-complete web chat including rooms, private messages, file uploads, presence tracking, and moderation tools. The server is a FastAPI app that uses async endpoints and WebSockets for real-time updates; persistent storage uses SQLite with WAL for concurrency. The project comes with Docker files and tests (pytest + Playwright).

Key features

  • Real-time messaging via WebSockets
  • Multi-device sessions and presence indicators
  • Rooms, private messaging, and basic moderation (mute/kick)
  • Secure JWT-based authentication and session management
  • File upload support with safe storage options
  • Tested with pytest and Playwright for end-to-end flows

Impact

  • Provides a small, deployable chat service suitable for demos, internal tooling, or as a starting point for production features.
  • Designed to be containerized and deployed with Docker; easy to scale horizontally behind a WebSocket-aware proxy.
  • Improves developer productivity with tests and clear API contracts for clients.

How to run locally

  1. Clone the repository: git clone https://github.com/mrprogrammingx/classic-web-based-online-chat-application.git
  2. Install dependencies and start the FastAPI server (uvicorn) with the appropriate environment variables.
  3. Run the WebSocket-capable server (uvicorn) and open the demo UI in a browser.

If you want, I can deploy a live demo for this chat project to your domain and attach a working WebSocket endpoint (will require deployment choices: small VM, Cloud Run, or a managed container service).
